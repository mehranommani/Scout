Architecture Overview

Scout — Company Intelligence Agent

LangGraph · FastAPI · Next.js 14 · Ollama · Langfuse · PostgreSQL · Qdrant

Agent Pipeline

  • • classify_input — LLM structured output (company / product / invalid)
  • • resolve_product — maps product name to owning company
  • • scrape_and_generate — 5 sources in parallel → LLM report
  • • validate_and_store — 4 structural checks, retry loop (max 3)
  • • Best attempt always persisted even on max retries

Data Sources

  • • Wikidata — SPARQL: founders, CEO, HQ, revenue
  • • DuckDuckGo — 3 targeted searches (financial, contact, general)
  • • OpenCorporates — legal registration, jurisdiction
  • • Crunchbase — funding rounds, investors, valuation
  • • LinkedIn — employee count, company overview

Infrastructure

  • • PostgreSQL — sessions, reports, eval scores
  • • Qdrant — vector search on report content
  • • Langfuse — LLM-as-judge eval, trace scores, spans
  • • Ollama — fully local LLM, no API keys required
  • • SSE — real-time agent progress streaming to UI
